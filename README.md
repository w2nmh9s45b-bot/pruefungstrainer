# Prüfungstrainer (PWA)

Interaktive Prüfungsschemata für das Verwaltungsstudium (HöV RLP) als Progressive Web App fürs iPhone. Die Startseite zeigt **Modul-Ordner** (Modul 2 „Rechtliche Grundlagen", Modul 5 „Besonderes Verwaltungsrecht"); darin liegen die **Fächer** (Modul 2: Allgemeines Verwaltungsrecht, Privatrecht, Staatsrecht/Europarecht · Modul 5: Baurecht, Kommunalrecht) und in diesen die Schemata zum Durchklicken – jeder Prüfungsschritt baut auf den vorherigen Antworten auf, mit Zurück-Funktion, Verlauf und kopierbarem Prüfungsweg. In den meisten Fächern sind die Schemata nach **Fachstudienjahren (FS 1–3)** gruppiert; Schemata, die in mehreren Jahren gebraucht werden, erscheinen in jeder betreffenden Gruppe.

## Schemata im Ordner Baurecht

1. **Anspruch auf Baugenehmigung, § 70 I 1 LBauO** – Vollprüfung (formell, § 61 LBauO, §§ 29–36 BauGB inkl. §§ 30/31/33/34/35, Einvernehmen § 36, Bauordnungsrecht, sonstiges öffentliches Recht)
2. **Maßnahmen der Bauaufsichtsbehörde, §§ 59, 80–82 LBauO** – Baueinstellung, Versiegelung, Nutzungsuntersagung, Beseitigung, Abbruch; formelle/materielle Illegalität, Störer, Ermessen (Bestandsschutz, Duldung, Art. 3 GG)
3. **Nachbarwiderspruch gegen die Baugenehmigung** – Nachbarbegriff, Schutznormtheorie, Fristen/Verwirkung, Gebietserhaltung, § 15 BauNVO, Rücksichtnahmegebot, Eilrechtsschutz (§ 212a BauGB, §§ 80a/80 V VwGO)
4. **Anspruch auf bauaufsichtliches Einschreiten** – Nachbar oder Gemeinde; Ermessensreduzierung auf Null, § 123 VwGO
5. **Wirksamkeit eines Bebauungsplans** – Verfahren, Fehlerfolgen §§ 214/215 BauGB, ergänzendes Verfahren, Erforderlichkeit, Planungsrahmen, Abwägung
6. **Plansicherung** – Veränderungssperre (§§ 14, 16–18 BauGB), Zurückstellung (§ 15), Vorkaufsrecht (§§ 24 ff.), Planreife (§ 33)

Inhaltliche Quellen: Obsidian-Vault „Brain“, `Studium/Studium 1-3 (MD)/Modul 5/Baurecht/` (Übersicht Anspruch auf Baugenehmigung, OLE-11-Skripte Zulässigkeit/Gebietsverträglichkeit, FS-III-Gesamtskript Birtel-Kaldenhoff 2021).

## Aufs iPhone bringen

Eine PWA braucht HTTPS. Der einfachste Weg ist kostenloses Hosting:

**Variante A – GitHub Pages (empfohlen):**
1. Diesen Ordner als Repository auf GitHub pushen (z. B. `pruefungstrainer`).
2. Repo → Settings → Pages → Branch `main`, Ordner `/ (root)` → Save.
3. Die Pages-URL (`https://<name>.github.io/pruefungstrainer/`) in Safari auf dem iPhone öffnen.
4. Teilen-Symbol → **„Zum Home-Bildschirm“**. Die App startet dann im Vollbild mit eigenem Icon und funktioniert dank Service Worker auch offline.

**Variante B – Netlify Drop:** Ordner auf https://app.netlify.com/drop ziehen, fertige URL in Safari öffnen, „Zum Home-Bildschirm“.

**Lokal testen (am Mac):**
```bash
cd "/Users/moritzschmidt/Brain/Apps/Pruefungstrainer"
python3 -m http.server 8741
# dann http://localhost:8741 im Browser
```

## Neues Schema oder neues Fach hinzufügen

1. Datei `schema-<name>.js` nach dem Muster der bestehenden Dateien anlegen (`window.SCHEMATA.push({...})`): `subject` bestimmt den Fach-Ordner (ein neuer `subject`-Wert erzeugt automatisch einen neuen Ordner auf der Startseite), `area` die Kategorie-Zeile auf der Karte, `fs: ["FS1"|"FS2"|"FS3", …]` ordnet das Schema optional einem oder mehreren Fachstudienjahren zu (dann gruppiert die Fach-Ansicht nach Jahren; mehrere Werte = Schema erscheint in mehreren Gruppen). Knoten (`frage`/`ergebnis`) mit Optionen, die per `next` auf den Folgeknoten zeigen; `set` setzt Flags; `routers` (`"@..."`) schalten abhängig von Flags; `verdictLabels` beschriftet die Ergebnis-Banner.
2. Die Datei in `index.html` vor `app.js` einbinden und in `sw.js` zu `ASSETS` ergänzen.
3. In `sw.js` die Versionsnummer in `CACHE` hochzählen, damit Geräte die neue Version laden.
4. Optional in `app.js` unter `SUBJECT_META` Icon und Beschreibung für das neue Fach hinterlegen.
